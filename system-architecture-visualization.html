üîó Key Layer Interactions
‚Üí
Trigger Management sends user identifiers to Identity & Profile Resolution
Before any outreach starts, the system must figure out who the user is and retrieve their profile
‚Üí
Agent Configuration Engine provides behavioral rules to Agent Framework
The agent gets its personality, knowledge selections, and execution rules from the config layer
‚Üí
Knowledge Base Integration provides merged knowledge payload to Agent Framework
Before conversation starts, all relevant knowledge (compliance, program, client, base) is assembled in priority order
‚Üí
Agent Framework sends conversation content to Outreach Channels Layer
The agent produces channel-agnostic content; channels layer handles formatting and delivery
‚Üí
Outreach Channels hands off to Channel Fallback Engine when delivery fails
If WhatsApp doesn't deliver, fallback engine automatically tries SMS, then Email
‚Üí
Agent Framework sends structured data to Document Template Layer
After conversation ends, captured data auto-fills compliance reports and care documentation
‚Üí
Agent Framework sends task completion to Journey Tracker
Outreach outcome (success, failure, escalation) advances the care journey to next step
‚Üí
Execution Logging feeds Outreach Scheduler with outcomes
Scheduler decides next action based on what happened: retry, schedule next, or stop
‚Üí
Consent Management blocks Outreach Channels if consent is missing
Every delivery attempt requires valid consent check. No consent = no send, period.
